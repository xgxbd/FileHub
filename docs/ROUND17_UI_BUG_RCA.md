# ROUND17 UI/下载问题根因分析（先分析后修复）

## 问题 1：页面切换时标题区域高度跳动

### 问题现象
- 在 `/files`、`/upload`、`/preview` 等页面切换时，标题区域视觉高度不稳定，用户感知为“页面头部在跳”。

### 可能原因
1. 顶部标题区域未强制单行显示，长标题在窄宽度下可能换行。
2. 卡片标题区（`p-card-title` + `p-card-subtitle`）未设置固定最小高度，不同页面文案长度导致内容起始位置变化。
3. 响应式下 `topbar` 在窄屏使用 `height: auto`，会放大上述波动。

### 验证步骤
1. 检查全局样式 `frontend/src/styles/main.css` 中 `topbar`、`page-info`、`p-card` 标题区域规则。
2. 检查路由元信息与页面标题文案长度（`frontend/src/router/index.js`）。
3. 对比各页面卡片标题与副标题文案长度，确认是否存在高度差来源。

### 验证结果
- `topbar` 在窄屏断点有 `height: auto` 规则。
- `page-info` 区域未固定单行截断规则。
- `p-card` 标题区域无固定最小高度，多个页面标题/副标题长度不同。

### 最终根因
- 头部信息区与页面卡片标题区缺少“固定高度约束”，在不同文案长度和不同断点下产生高度波动。

### 修复方案
1. 对 `page-info` 添加固定高度与单行省略。
2. 对 `p-card` 标题区域设置最小高度与稳定行高。
3. 在窄屏保持头部稳定高度，避免路由切换触发高度变化。

## 问题 2：下载失败（你现场反馈）

### 问题现象
- 点击下载按钮后出现下载失败。

### 可能原因
1. 权限错误（401/403）。
2. 文件不存在/已删除（404）。
3. Range 范围异常（416）。
4. 响应头构造异常导致后端抛错（500）。

### 验证步骤
1. 使用后端 `TestClient` 复现完整链路：注册 -> 登录 -> 上传中文文件名 -> 下载。
2. 捕获异常栈，定位失败点。
3. 对照下载接口实现（`backend/app/api/files.py`）确认响应头拼装逻辑。

### 验证结果
- 复现命令触发异常：`UnicodeEncodeError: 'latin-1' codec can't encode characters...`。
- 报错位置：`backend/app/api/files.py` 下载接口返回 `Response` 时，`Content-Disposition` 使用原始中文文件名。

### 最终根因
- 下载响应头 `Content-Disposition` 直接写入中文文件名，Starlette 在 latin-1 编码响应头时抛异常，导致下载失败。

### 修复方案
1. 下载接口使用 RFC 5987 编码：`filename*=UTF-8''...`。
2. 同时保留 ASCII fallback 的 `filename="..."`。
3. 补充自动化测试：中文文件名上传后下载应成功。
