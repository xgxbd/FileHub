# FileHub V0.1 开发任务拆分（优先级）

## P0 项目基线与治理

1. 初始化 GitFlow 基线分支（`main/develop/feature/*`）。
2. 建立目录骨架（`frontend/`、`backend/`、`infra/`、`docs/`）。
3. 建立 `docs/PROGRESS.md` 固定记录模板。
4. 明确后端虚拟环境规范，禁止系统 Python 与全局依赖。
5. 建立 Docker Compose 基础服务（MySQL/Redis/MinIO）。

## P1 产品与 UI 先行

1. 固化页面信息架构与关键流程（上传、下载、回收、审计）。
2. 生成 5 套高保真 UI 方案（同页面集，不同视觉风格）并提供预览图。
3. 选定最终 UI 后冻结设计令牌与组件规范。

## P2 后端 MVP 能力

1. 认证模块（注册/登录/刷新/权限中间件）。
2. 文件元数据模型与对象存储抽象。
3. 分片上传链路（创建任务、分片上传、合并、断点续传）。
4. Range 下载链路。
5. 回收站与操作日志。
6. 管理端基础管理接口。

## P3 前端 MVP 能力

1. 登录注册与会话管理。
2. 文件中心（列表、上传、进度、断点续传下载）。
3. 预览页（图片/PDF/TXT）。
4. 回收站页面。
5. 管理端最简可用版（文件管理 + 日志审计）。

## P4 测试与发布准备

1. 后端单元/集成测试（上传、下载、权限、回收站、日志）。
2. 前端关键流程 E2E（登录、上传、下载、恢复、删除）。
3. 性能与稳定性基线（大文件、并发、重试）。
4. 发布清单与回滚预案。

## 依赖顺序（必须遵守）

- P0 完成后进入 P1。
- UI 未确认前，不进入 P2/P3 实现。
- P2/P3 完成后再进入 P4。
