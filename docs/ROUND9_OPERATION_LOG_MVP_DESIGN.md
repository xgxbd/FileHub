# Round 9 操作日志 MVP 设计

## 1. 目标

- 实现关键操作留痕与管理员可查询能力，覆盖“登录、上传、下载、删除、恢复、彻底删除”主链路。

## 2. 范围

- 后端：
  - 新增操作日志数据模型 `operation_logs`
  - 关键接口写入日志记录（成功行为）
  - 管理员日志查询接口：`GET /admin/logs`
- 前端：
  - 新增日志页 `/admin/logs`
  - 支持按操作类型、用户ID、时间分页查询

## 3. 数据策略

- 表字段（MVP）：
  - `id`、`user_id`、`username_snapshot`、`action`、`target_type`、`target_id`
  - `detail_json`（JSON 字符串）
  - `created_at`
- 日志写入策略：
  - 仅记录成功操作，失败链路后续迭代补充
  - 不阻塞主流程：日志写入失败不影响主接口成功返回

## 4. 权限策略

- 普通用户：不能访问管理员日志查询接口
- 管理员：可查看全量操作日志
