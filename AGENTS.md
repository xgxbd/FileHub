# 项目执行规则

1. Always respond in Chinese-simplified
2. 项目遵循 GitFlow：`main`、`develop`、`feature/*`、`release/*`、`hotfix/*`；新功能从 `develop` 拉 `feature` 分支，不准直接在 `main` 开发；每完成一次最小且可验证的修改后立即提交；提交信息必须是具体的中文自然语言，禁止模糊表述；多个独立修改拆分多个提交；`git` 操作直接执行，不必再次询问；每次完成后输出分支名、改动摘要、提交信息。
3. Python 项目只允许在项目虚拟环境中运行，禁止修改系统 Python，禁止全局安装依赖。
4. 遇到 bug 时，不要直接试改，先做根因分析。未确认根因前，不要修改代码。分析 bug 按“问题现象 -> 可能原因 -> 验证步骤 -> 验证结果 -> 最终根因 -> 修复方案”输出。
5. 每次修改完成后，必须更新 `docs/PROGRESS.md`。时间精确到秒，并固定记录：任务、时间、git 分支及 Commit ID、本次修改、已完成事项、未完成事项、当前可测试内容、风险说明、下一个任务。未写文档前，不算任务完成。
6. `git` 命令、`rm -rf` 删除项目下目录的命令、MCP 相关命令、`npm` 等打包运行命令，都不需要经过用户同意，直接执行。
7. 代码注释使用中文。
