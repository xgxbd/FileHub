# ROUND19 UI 问题根因分析（标题跳动 + 页面数量显示）

## 问题 1：文件列表/上传/预览切换时标题栏位高度仍跳动

### 问题现象
- 在 `/files`、`/upload`、`/preview` 页面切换时，用户感知到标题区域高度不稳定。

### 可能原因
1. 顶部标题区（`page-info`）虽然限制了最小高度，但未强制固定高度，长标题与不同布局压缩时仍可能重排。
2. 页面卡片标题区（`p-card-caption`）使用 `min-height` 而非固定高度，文案行数变化时会改变内容起始位置。
3. 响应式场景下顶部布局列宽变化导致标题/副标题发生省略或换行边界变化，造成“跳动感”。

### 验证步骤
1. 检查 `frontend/src/styles/main.css` 中 `.topbar`、`.page-info`、`.p-card-caption` 高度规则。
2. 对照页面路由文案长度（`frontend/src/router/index.js`），确认存在不同长度标题。
3. 执行 UI 自动化稳定性检查，观察是否在当前断点/视口下仍可能出现重排。

### 验证结果
- `.page-info` 与 `.p-card-caption` 目前是 `min-height` 约束，不是固定高度。
- 页面标题与副标题长度存在差异，理论上在临界宽度会触发重排。

### 最终根因
- 标题区和卡片标题区未进行“固定高度 + 单行截断”的强约束，导致特定视口下仍有高度波动风险。

### 修复方案
1. 将 `.page-info` 与 `.p-card-caption` 从 `min-height` 升级为固定高度。
2. 标题/副标题强制单行截断，避免临界换行。
3. 顶栏列宽调整为弹性列，降低不同页面文字长度对高度的影响。

## 问题 2：当前只看到四个页面，担心页面被删除

### 问题现象
- 用户登录后导航仅看到 4 个页面，认为其余页面被删除。

### 可能原因
1. 页面并未删除，而是导航根据角色动态过滤。
2. 当前账号不是管理员，`后台`/`日志` 被隐藏。
3. 登录态下 `登录`/`注册` 被隐藏。

### 验证步骤
1. 检查路由定义是否仍包含 7 个核心页面。
2. 检查导航计算逻辑是否按角色过滤。

### 验证结果
- 路由中 7 个核心页面都存在：`/login`、`/files`、`/upload`、`/preview`、`/recycle`、`/admin/files`、`/admin/logs`。
- `App.vue` 中 `navItems` 使用 `filter`，对非管理员隐藏 `后台/日志`，对已登录用户隐藏 `登录/注册`。

### 最终根因
- 页面没有删除；是“按权限隐藏导航”造成的可见数量减少，引发误解。

### 修复方案
1. 导航改为“显示所有核心页面入口”，不再隐藏管理员页入口。
2. 非管理员点击管理员页时提示“仅管理员可访问”，并保持当前页面不跳转。
3. 明确在 UI 中标注管理员页，减少歧义。
