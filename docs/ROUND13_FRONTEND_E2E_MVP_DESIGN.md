# Round 13 前端关键流程 E2E MVP 设计

## 1. 目标

- 补齐前端关键流程自动化回归能力，覆盖登录、上传、下载、删除、恢复主链路。

## 2. 范围

- 测试框架：Playwright
- 测试对象：Web 前端（配合本地后端 API）
- 覆盖场景：
  - 用户注册/登录进入文件中心
  - 文件上传后出现在列表
  - 文件下载动作可触发下载事件
  - 文件删除后可在回收站看到
  - 回收站恢复后返回文件中心可见

## 3. 执行策略

- 在测试配置中同时启动：
  - 后端服务（FastAPI + 本地 SQLite）
  - 前端服务（Vite）
- 浏览器使用 Chromium（Playwright）
- 使用唯一用户名（时间戳）避免数据冲突

## 4. 最小提交拆分

### 提交 A（设计）
- 文档 + 进度记录
- 验证：文档可评审

### 提交 B（E2E 基建）
- 引入 Playwright 依赖与配置
- 新增基础启动脚本与 npm 命令
- 验证：`npx playwright test --list` 可列出用例

### 提交 C（关键流程用例）
- 实现登录/上传/下载/删除/恢复 E2E 用例
- 验证：`npm run e2e` 通过

### 提交 D（联调稳定性补充）
- 补充必要的等待策略与最小重试
- 验证：连续执行两次通过

### 提交 E（文档收口）
- 更新前后端 README 与进度文档
- 验证：按文档可在本地执行 E2E
