# ROUND23 文件列表页树形区与查询区调整设计

## 目标

- 左侧文件树更接近 VS Code 资源管理器的阅读密度。
- 查询区从“最小字节/最大字节”改为“关键字 + 排序”。
- 当前目录显示必须与树节点当前选中目录一致。

## 范围

- 仅调整文件列表页 `/files`。
- 不修改上传、预览、回收站、后台管理页的查询表单。

## 设计决策

### 1. 左侧文件树

- 左侧树区域宽度从当前窄列增大到约 `320px`。
- 字号降到 `12px`，行高压缩，接近 VS Code 侧栏密度。
- 树节点内容缩小内边距，让同屏可见目录更多。
- 保留当前“根目录 / 刷新”按钮，不增加手动输入。

### 2. 查询区

- 保留“文件名关键字”。
- 删除“最小大小（字节）/最大大小（字节）”两个字段。
- 新增“排序方式”字段，MVP 支持：
  - 最新上传
  - 最早上传
  - 文件名 A-Z
  - 文件名 Z-A
  - 文件大小从大到小
  - 文件大小从小到大

### 3. 当前目录显示

- 顶部“当前目录”标签直接由 `selectedDirectory` 驱动。
- 根目录显示为 `根目录 /`。
- 点击树节点后，标签文本与文件列表查询参数同步更新。

## 实施拆分

1. 后端：`GET /files` 增加 `sort_by` 参数与排序测试。
2. 前端：文件列表页去掉大小过滤，换为排序控件；树区域增宽并压缩字体。
3. 验证：后端 `pytest tests/test_file_list_api.py`，前端 `npm run build`。
