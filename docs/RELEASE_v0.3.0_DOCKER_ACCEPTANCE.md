# v0.3.0 Docker 最小人工验收清单

## 验收目标

确认 `release/v0.3.0` 在 Docker 一键启动形态下可正常提供完整 Web 文件仓库能力。

## 启动命令

```bash
cd /Users/xloser/Owner/Code/VibeCoding/FileHub/infra
docker compose up -d --build
```

## 基线检查

1. 页面入口可访问：`http://127.0.0.1:8000/files`
2. 健康检查可访问：`http://127.0.0.1:8000/api/healthz`
3. MinIO Console 可访问：`http://127.0.0.1:9001`
4. 管理员账号可登录：
   - 用户名：`admin`
   - 密码：`ChangeMe123!`

## 最小人工验收步骤

### 1. 登录验收

1. 打开 `http://127.0.0.1:8000/login`
2. 使用管理员账号登录
3. 预期：成功进入文件列表页

### 2. 文件夹验收

1. 进入文件列表页
2. 新建文件夹 `logs`
3. 进入 `logs`
4. 预期：顶部显示 `当前目录：/logs/`

### 3. 上传验收

1. 进入上传页
2. 选择两个 `txt` 文件，或一次拖入两个 `txt` 文件
3. 点击开始上传
4. 预期：
   - 两个文件都进入上传队列
   - 上传完成后状态为成功
   - 返回文件列表后能在 `logs` 下看到文件

### 4. 预览验收

1. 进入预览页
2. 选择刚上传的 `txt`
3. 预期：
   - 可看到文本内容
   - 右侧最近文件可切换

### 5. 下载验收

1. 返回文件列表
2. 下载一个刚上传的 `txt`
3. 预期：浏览器正常下载文件

### 6. 回收站验收

1. 在文件列表删除刚上传的一个文件
2. 进入回收站
3. 预期：可看到被删除文件
4. 点击恢复
5. 预期：文件恢复成功，回文件列表后可再次看到

### 7. 管理端验收

1. 进入后台管理页
2. 搜索一个刚上传的文件名
3. 预期：可看到文件记录与目录
4. 进入日志页
5. 搜索 `upload` 或 `soft_delete`
6. 预期：能看到对应日志

## 通过标准

满足以下条件即可判定 `v0.3.0` 可进入正式发布：

1. Docker 一键启动成功
2. 登录成功

4. 预览成功
5. 下载成功
6. 回收站恢复成功
7. 管理页与日志页可用
