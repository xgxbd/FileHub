# 页面信息架构与关键流程（P1）

## 页面信息架构

- 认证域
  - 登录页
  - 注册页
- 用户域
  - 文件中心页
  - 文件详情/预览页
  - 回收站页
  - 个人设置页
- 管理域
  - 管理仪表页
  - 文件管理页
  - 日志审计页
  - 用户管理页（最简）
- 错误域
  - 401/403/404

## 关键流程

### 1. 上传流程（分片）

1. 用户在文件中心选择文件并触发上传。
2. 前端计算文件哈希并请求创建上传任务。
3. 后端返回已上传分片列表，前端补传缺失分片。
4. 分片上传完成后请求合并。
5. 后端校验成功后写入元数据并返回文件记录。

### 2. 下载流程（断点续传）

1. 用户点击下载，前端请求文件下载地址。
2. 浏览器发起支持 Range 的下载请求。
3. 网络中断后继续下载时携带 Range 头从断点续传。

### 3. 删除与回收流程

1. 用户删除文件，系统执行软删除并写入日志。
2. 文件进入回收站，可恢复到原目录。
3. 管理员可执行彻底删除，触发对象存储物理删除。

### 4. 审计流程

1. 关键行为统一写入操作日志。
2. 管理员可按用户、操作类型、时间范围查询。
