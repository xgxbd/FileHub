# FileHub V0.1 精简 PRD

## 1. 项目目标

- 构建可自托管的 Web 文件仓库系统，形成上传、管理、下载、回收、审计闭环。
- 首版以稳定可用为目标，优先覆盖已确认的 7 类核心能力，避免过度设计。

## 2. 目标用户

- 普通用户：上传、下载、预览、回收站操作。
- 管理员：文件后台管理、操作日志审计、异常追踪。

## 3. 核心功能（V0.1）

- 用户登录注册：账号密码 + JWT 会话。
- 大文件分片上传：分片上传、断点续传、哈希秒传校验。
- 文件下载断点续传：支持 HTTP Range。
- 文件后台管理：列表筛选、状态管理、批量操作。
- 文件预览：图片/PDF/文本可预览，其他类型仅下载。
- 回收站：软删除、恢复、彻底删除（管理员可见）。
- 用户操作日志：登录、上传、下载、删除、恢复等行为留痕。

## 4. 页面结构

- 认证：登录页、注册页。
- 用户端：文件中心、文件详情/预览、回收站。
- 管理端：仪表页、文件管理页、日志审计页、用户管理页（最简）。
- 公共：个人设置、401/403/404 页面。

## 5. MVP 范围

- 单租户场景。
- 账号密码登录，不做 SSO/OAuth。
- 仅用户/管理员两级角色，不做细粒度 RBAC。
- 基础检索（文件名、时间、大小），不做全文检索。
- 预览仅支持图片/PDF/TXT。

## 6. 暂不实现

- 在线协作编辑
- 外链分享
- 版本历史
- 病毒扫描
- 消息通知
- 多区域存储
- 工作流审批

## 7. 技术方案建议

- 前端：Vue 3 + PrimeVue + Pinia + Vue Router。
- 后端：FastAPI + SQLAlchemy + Alembic + JWT。
- 数据与存储：MySQL + Redis + MinIO。
- 基础设施：Docker Compose（MySQL/Redis/MinIO/API/Web）。
- 可观测：结构化日志 + 请求 ID + 关键接口耗时统计。
- 安全基线：密码哈希、接口鉴权、文件名与路径安全校验、上传策略、下载鉴权。

## 8. 风险与控制

- 风险：范围失控导致首版延期。  
  控制：冻结 MVP 范围，超出项统一放入后续版本。
- 风险：大文件链路返工。  
  控制：先定义上传/下载协议和状态机，再开发具体实现。
- 风险：权限模型复杂化。  
  控制：首版固定双角色，后续按里程碑扩展。
