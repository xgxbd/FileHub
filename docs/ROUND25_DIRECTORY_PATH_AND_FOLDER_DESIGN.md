# 第二十五轮开发：目录路径语义与文件夹能力设计

## 一、问题一根因分析：当前目录显示语义不准确

- 问题现象：
  - 点击左侧 `logs` 目录后，页面顶部显示的是 `当前目录：logs`。
  - 这不符合文件树的绝对路径语义，用户无法立刻判断它是根目录下的 `logs`，还是更深层级中的 `logs`。

- 可能原因：
  - 当前前端把目录状态保存为相对路径片段，例如 `logs`、`docs/specs`。
  - 显示层直接把该值原样展示，没有统一转换成绝对路径格式。

- 验证步骤：
  - 查看 `frontend/src/views/FileCenterView.vue` 中 `currentDirectoryLabel` 的计算逻辑。
  - 对比当前目录树数据结构和后端 `/files?directory=` 的筛选语义。

- 验证结果：
  - 根目录使用 `__root__` 标记。
  - 非根目录直接显示 `selectedDirectory` 原始值，没有补齐开头和结尾的 `/`。

- 最终根因：
  - 目录内部状态与用户展示语义混用，缺少统一的“展示路径格式化”层。

- 修复方案：
  - 保持内部筛选值继续使用 `__root__` / `logs` / `docs/specs`，避免影响查询参数。
  - 新增展示格式化规则：
    - 根目录显示 `/`
    - 一级目录显示 `/logs/`
    - 多级目录显示 `/docs/specs/`

## 二、问题二实施设计：支持任意目录创建/删除文件夹

## 1. 现状结论

- 当前文件树来自 `file_objects.file_name` 的路径反推。
- 这种方式只能表示“已有文件所在目录”，不能表示空文件夹。
- 因此以下能力当前无法可靠支持：
  - 在任意目录新建空文件夹
  - 删除一个空文件夹
  - 在没有文件的情况下仍然显示该目录

## 2. MVP 设计原则

- 不引入复杂的目录权限继承。
- 不做文件夹回收站。
- 不做递归删除，先只允许删除空文件夹。
- 不改动现有文件对象存储结构，文件内容仍只放 MinIO/本地对象目录。

## 3. 数据模型设计

- 新增表：`folders`
- 字段：
  - `id`
  - `owner_id`
  - `path`
  - `name`
  - `parent_path`
  - `is_deleted`
  - `created_at`
  - `updated_at`

- 约束：
  - 同一用户下 `path` 唯一
  - `path` 采用相对目录格式保存，如：
    - 根目录下 `logs`
    - 二级目录 `docs/specs`
  - 根目录本身不落表

## 4. 接口设计

- `GET /folders/tree`
  - 返回当前用户目录树
  - 目录来源 = 显式文件夹 + 文件路径反推目录
  - 这样既兼容历史数据，也支持新建空目录

- `POST /folders`
  - 入参：
    - `parent_directory`
    - `folder_name`
  - 规则：
    - 允许根目录下创建
    - 自动标准化路径，拒绝 `..`、空白、重复分隔符
    - 同级重名直接报错

- `DELETE /folders`
  - 入参：
    - `path`
  - 规则：
    - 只允许删除空目录
    - 若目录下存在未删除文件，拒绝删除
    - 若目录下存在子目录，拒绝删除

## 5. 前端交互设计

- 页面位置：文件列表页左侧文件树头部
- 新增按钮：
  - `新建文件夹`
  - `删除当前文件夹`

- 交互：
  - 点击 `新建文件夹`
    - 在当前目录下弹出输入框
    - 输入成功后刷新目录树并自动选中新目录
  - 点击 `删除当前文件夹`
    - 根目录不可删除
    - 非空目录按钮仍显示，但点击后给出明确失败原因

## 6. 测试设计

- 后端测试：
  - 根目录新建文件夹成功
  - 子目录下新建文件夹成功
  - 同级重名失败
  - 删除空目录成功
  - 删除含文件目录失败
  - 删除含子目录目录失败

- 前端/E2E：
  - 创建 `logs`
  - 创建 `/logs/archive/`
  - 点击目录后顶部显示 `/logs/`、`/logs/archive/`
  - 删除空目录成功
  - 删除非空目录失败并展示错误信息
