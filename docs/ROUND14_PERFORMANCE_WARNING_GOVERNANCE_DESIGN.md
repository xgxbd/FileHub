# Round 14 性能与告警治理 MVP 设计

## 1. 目标

- 消除当前已知关键告警并降低前端首包体积，提升发布质量可维护性。

## 2. 范围

- 后端告警治理：
  - 将 FastAPI `on_event("startup")` 迁移为 `lifespan`
  - 将 `HTTP_416_REQUESTED_RANGE_NOT_SATISFIABLE` 替换为新常量
- 前端性能治理：
  - 路由组件改为懒加载，减少主包体积
  - 必要时配合 Vite `manualChunks` 拆包

## 3. 非目标

- 不做架构级缓存重构
- 不做对象存储重试机制重写
