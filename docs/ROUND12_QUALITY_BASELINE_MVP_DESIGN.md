# Round 12 质量基线与发布准备 MVP 设计

## 1. 目标

- 建立最小可执行的发布前质量基线，覆盖“关键流程可回归 + 基础性能观测 + 发布回滚文档”。

## 2. 范围

- 后端：
  - 新增烟雾验证脚本：覆盖注册/登录/上传/下载/删除/恢复主链路
  - 新增性能基线脚本：统计核心接口平均耗时（轻量基线）
- 文档：
  - 发布清单（发布前检查项）
  - 回滚预案（失败判定、回滚步骤、数据风险说明）

## 3. 不在本轮范围

- 真实压测平台（k6/JMeter）接入
- 完整浏览器 E2E 自动化（本轮保留为后续增强）

## 4. 最小提交拆分

### 提交 A（设计）
- 文档 + 进度记录
- 验证：文档可评审

### 提交 B（烟雾验证脚本）
- 新增后端主流程 smoke 脚本
- 验证：脚本返回成功结果

### 提交 C（性能基线脚本）
- 新增轻量性能统计脚本（多次请求平均耗时）
- 验证：脚本输出可读基线指标

### 提交 D（发布清单与回滚预案）
- 新增发布与回滚文档
- 验证：文档可执行、步骤清晰

### 提交 E（文档收口）
- 更新 README 与进度文档
- 验证：按文档可执行 smoke + baseline + release checklist
