# Round 15 认证哈希告警治理设计

## 1. 目标

- 消除测试中由 `passlib` 间接触发的 `crypt` 弃用告警。
- 保持现有用户密码校验兼容，不影响登录能力。

## 2. 范围

- 后端认证加密实现：
  - 密码哈希与校验从 `passlib` 迁移到 `bcrypt` 原生接口
  - 保持哈希格式兼容（`$2b$...`）
- 依赖治理：
  - 移除 `passlib[bcrypt]` 依赖
  - 保留 `bcrypt`

## 3. 兼容策略

- 新用户注册：使用 `bcrypt.hashpw` 生成哈希
- 老用户登录：继续使用 `bcrypt.checkpw` 校验已有 `bcrypt` 哈希
- 不做批量密码重哈希迁移
