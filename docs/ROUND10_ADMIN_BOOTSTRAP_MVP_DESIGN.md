# Round 10 管理员初始化 MVP 设计

## 1. 目标

- 提供可重复、可配置的管理员初始化能力，解决“管理员功能可用性依赖手工改库”的问题。

## 2. 范围

- 后端：
  - 新增环境变量：`ADMIN_BOOTSTRAP_ENABLED`、`ADMIN_EMAIL`、`ADMIN_USERNAME`、`ADMIN_PASSWORD`
  - 启动时检查并创建/升级管理员账号（幂等）
  - 管理员账号已存在时仅校正角色，不覆盖密码
- 前端：
  - 不改页面，仅依赖现有登录流程进入管理员能力

## 3. 规则

- 仅在 `ADMIN_BOOTSTRAP_ENABLED=true` 时执行
- 创建规则：
  - 按 `ADMIN_EMAIL` 或 `ADMIN_USERNAME` 匹配用户
  - 不存在则创建新用户（`role=admin`）
  - 存在则仅确保 `role=admin`
- 安全约束：
  - 默认关闭开关
  - 默认占位密码需在部署时覆盖
