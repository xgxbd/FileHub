FileHub 方案 D 可视化标注图

范围：登录、文件列表、上传、预览、回收站、后台管理、操作日志。每个数字标注对应右侧说明。

测试建议：按页面从上到下执行 1~N 号操作。每项都应出现“预期反馈”，若不符合即可判定问题。

1. 登录页

/login
1 2 3 4

可点击项

  1. 账号输入框：输入用户名或邮箱。
  2. 密码输入框：输入密码。
  3. 登录：成功跳转文件列表；失败显示错误消息。
  4. 注册：跳转注册页。

2. 文件列表页

/files
1 2 3 4 5 6 7 8

可点击项

  1. 文件列表导航：切回文件列表。
  2. 上传导航：进入上传页。
  3. 上传文件：跳转 `/upload`。
  4. 批量下载：下载选中文件（未选中时无动作）。
  5. 筛选：触发文件列表查询。
  6. 预览：跳转 `/preview/:fileId`。
  7. 下载：触发浏览器下载。
  8. 删除：弹确认，确认后进入回收站。

3. 上传页

/upload
1 2 3

可点击项

  1. 文件选择框：选择待上传文件。
  2. 开始上传：创建会话 -> 分片上传 -> 合并完成。
  3. 返回文件列表：跳转 `/files`。

4. 预览页

/preview/:fileId?
1 2 3 4

可点击项

  1. 文件ID输入：输入要预览的文件 ID。
  2. 加载预览：请求预览接口并展示内容。
  3. 返回文件列表：跳转 `/files`。
  4. 下载：下载当前文件。

5. 回收站页

/recycle
1 2 3 4 5 6

可点击项

  1. 仅看我删除：过滤本人删除记录。
  2. 最近30天：按时间过滤。
  3. 恢复所选：恢复选中记录。
  4. 彻底删除：不可恢复删除。
  5. 行内恢复：恢复当前行。
  6. 行内彻底删除：彻底删除当前行。

6. 后台管理页

/admin/files
1 2 3

可点击项

  1. 用户管理：进入用户管理（后续扩展）。
  2. 存储策略：进入存储策略（后续扩展）。
  3. 导出报表：导出后台统计报表。

7. 操作日志页

/admin/logs
1 2 3 4

可点击项

  1. 用户筛选：按用户过滤日志。
  2. 操作类型筛选：按上传/下载/删除过滤。
  3. 时间筛选：按时间范围过滤。
  4. 查询：执行筛选刷新日志列表。
详细行为与异常反馈请配合文档：`docs/UI_INTERACTION_MAP_SCHEME_D.md`。
