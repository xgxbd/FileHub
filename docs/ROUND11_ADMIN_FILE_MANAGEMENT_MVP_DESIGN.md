# Round 11 管理端文件管理 MVP 设计

## 1. 目标

- 提供管理员专属文件管理页，实现全量文件查询与基础状态操作（删除/恢复/彻底删除）。

## 2. 范围

- 后端：
  - 新增管理员文件查询接口：`GET /admin/files`
  - 支持筛选：`owner_id`、`keyword`、`status`（active/deleted）、分页
  - 普通用户访问返回 `403`
- 前端：
  - 新增页面 `/admin/files`
  - 展示全量文件列表（含 owner_id）
  - 针对状态提供操作按钮：
    - `active`：删除（软删除）
    - `deleted`：恢复、彻底删除

## 3. 设计取舍（MVP）

- 不做批量操作与高级策略
- 不做复杂审批流
- 复用现有删除/恢复/彻删接口，避免重复后端逻辑
