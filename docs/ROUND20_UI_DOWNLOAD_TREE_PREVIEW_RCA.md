# ROUND20 问题根因分析（标题稳定、下载500、交互可见性、文件树、预览路径）

## 问题 1：页面切换时标题栏高度不一致

- 问题现象：在“文件列表、上传、预览、回收站、后台日志”间切换时，标题区域视觉高度有跳动。
- 可能原因：
  - 顶栏与卡片标题区存在最小高度而非刚性高度，在不同内容长度或容器宽度下发生重排。
  - 右上角控件数量、宽度变化导致中间标题可用宽度变化。
- 验证步骤：
  - 检查 `frontend/src/styles/main.css` 中 `.topbar`、`.page-info`、`.p-card .p-card-caption` 高度约束是否为固定值。
  - 检查 `frontend/src/App.vue` 顶栏右侧控件在不同页面是否有显隐差异。
- 验证结果：
  - 样式存在 `height + min-height` 混用，仍有在临界宽度下触发重排的空间。
  - 顶栏右侧使用按钮样式控件，尺寸与视觉权重较高，放大了“跳动感”。
- 最终根因：标题区固定策略不够严格，且右上角控件视觉干扰过强。
- 修复方案：
  - 将顶栏与标题容器改为刚性高度（固定 `height`），并统一标题行高/截断策略。
  - 右上角改为明确的状态标签或真实可点击行为，避免伪按钮造成布局和认知抖动。

## 问题 2：点击下载后报 HTTP 500

- 问题现象：下载按钮转圈后失败，前端提示 HTTP 500。
- 可能原因：
  - 数据库记录存在，但 MinIO/本地对象文件缺失，读取时抛异常未被接口层转换。
  - Range 参数边界处理异常。
- 验证步骤：
  - 检查 `backend/app/api/files.py` 下载接口异常处理逻辑。
  - 检查 `backend/app/services/object_storage.py` 对象不存在时抛出异常类型。
- 验证结果：
  - 对象不存在时会抛 `FileNotFoundError`。
  - 下载接口未捕获该异常，最终变为 500。
- 最终根因：对象缺失异常未在 API 层转换为业务可理解状态码。
- 修复方案：
  - 下载与预览接口统一捕获 `FileNotFoundError`，返回 404（文件内容不存在）。
  - 增加回归测试覆盖该分支。

## 问题 3：右上角“全局搜索/管理员”看起来可点击但无功能

- 问题现象：鼠标悬停有按钮反馈，但点击后无业务动作，造成误导。
- 可能原因：
  - 使用了 `Button` 组件但未绑定行为。
- 验证步骤：
  - 检查 `frontend/src/App.vue` 顶栏动作区组件类型与点击事件。
- 验证结果：
  - 两个元素均为按钮渲染，未绑定业务逻辑。
- 最终根因：占位设计未区分“状态信息”和“可操作按钮”。
- 修复方案：
  - 改为 `Tag/Chip` 状态信息（不可点击），并在页面交互说明中标注用途。

## 问题 4：文件列表需要文件树（多级目录）

- 问题现象：当前仅平铺文件列表，不支持目录视角。
- 可能原因：
  - 现有模型仅存 `file_name`，前端未解析路径层级。
  - 上传页未提供“目标目录”输入。
- 验证步骤：
  - 检查上传请求中 `file_name` 是否可携带目录前缀。
  - 检查文件列表页是否有目录树组件。
- 验证结果：
  - 上传接口允许 `file_name` 为路径字符串（如 `docs/a.txt`）。
  - 文件列表页未提供树形目录视图。
- 最终根因：能力已具备（路径字符串）但前端未产品化为目录树交互。
- 修复方案：
  - 上传页新增“目标文件夹”输入，提交时组装为 `folder/filename`。
  - 文件列表页基于 `file_name` 构建树形目录，支持点击目录过滤。
  - MVP 先实现“基于已有文件路径”的虚拟文件树，不引入独立文件夹表。

## 问题 5：预览不应依赖 fileId，且 TXT 预览失败

- 问题现象：预览页要求手输 fileId，不符合使用习惯；部分 TXT 无法预览。
- 可能原因：
  - 预览页路由和交互以 `fileId` 为中心，而不是“从文件列表选择”。
  - 预览能力仅按 MIME 判断，`application/octet-stream` 的 `.txt` 文件被拦截。
- 验证步骤：
  - 检查 `frontend/src/views/PreviewView.vue` 交互入口。
  - 检查 `backend/app/api/files.py::_is_previewable` 规则。
- 验证结果：
  - 页面主入口为手输 fileId。
  - 预览判断仅支持 `text/*`、`image/*`、`application/pdf`。
- 最终根因：预览交互与用户心智不一致；预览判定缺少扩展名兜底。
- 修复方案：
  - 预览页改为“文件列表选择 + 路由参数携带 fileId/fileName”，移除手输 ID 主路径。
  - 后端预览规则增加文本扩展名兜底（`.txt/.md/.log/.json/.csv`）。

## 实施顺序（最小提交）

3. 前端增强：上传目标文件夹 + 文件列表树形目录过滤 + 预览页去 fileId 手输。
4. 文档收口：更新交互说明与 `docs/PROGRESS.md`。
