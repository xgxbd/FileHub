# 第二十七轮开发：文件夹生命周期与文件树布局改造

## 一、Bug 根因分析：非空目录删除后从文件树消失

- 问题现象：
  - 用户删除 `logs` 目录时，界面出现“目录下仍有文件，不能删除”一类提示。
  - 但随后左侧文件树里看不到 `logs`，用户感知上像“非空目录还是被删掉了”。

- 可能原因：
  - 可能是后端错误地允许删除了非空目录。
  - 可能是前端在失败分支里仍然主动把当前目录切回上级目录并刷新树。
  - 可能是文件树只展示活跃目录，不展示已删除目录或仅包含回收站文件的目录，导致目录被隐藏而不是被真正删除。

- 验证步骤：
  - 检查 `backend/app/services/folder_service.py` 中 `delete_folder()` 的删除判定。
  - 检查 `frontend/src/views/FileCenterView.vue` 中 `removeCurrentFolder()` 的成功 / 失败分支。
  - 检查 `list_directory_paths()` 的目录树来源，确认是否只聚合活跃文件与未删除显式文件夹。

- 验证结果：
  - `delete_folder()` 在 `recursive=false` 语义下，遇到活跃文件会直接报错，不会继续执行删除。
  - `removeCurrentFolder()` 仅在接口成功返回后才切回上级目录，失败分支不会主动移除节点。
  - `list_directory_paths()` 当前只基于：
    - `Folder.is_deleted = false` 的显式文件夹
    - `FileObject.is_deleted = false` 的活跃文件路径
  - 因此目录一旦被标记删除，或目录下只剩回收站文件，该目录就不会出现在左侧树。

- 最终根因：
  - 目录删除判定本身没有放过活跃文件。
  - 真正的问题是“目录树可见性”和“目录回收站”没有打通：
    - 左侧文件树只看活跃目录
    - 回收站只支持文件，不支持文件夹
    - 所以用户看见的是目录消失，但没有地方恢复或核对该目录状态

- 修复方案：
  - 补齐文件夹完整生命周期：
    - 文件夹重命名
    - 递归删除非空文件夹
    - 文件夹回收站（列表 / 恢复 / 彻底删除）
  - 删除非空目录时改为“递归移入回收站”，而不是硬拦截到无法处理。
  - 回收站新增文件夹视图，用户能看到已删除目录并恢复。
  - 左侧文件树继续只展示活跃目录，但回收站能承接目录状态，不再出现“看不到也恢复不了”的断层。

## 二、MVP 实施设计

## 1. 文件夹重命名

- 新接口：`PATCH /folders/rename`
- 入参：
  - `path`
  - `new_name`
- 行为：
  - 仅修改目录名称，不改变层级
  - 级联更新：
    - 目标目录及其子目录的 `Folder.path` / `parent_path` / `name`
    - 目录下所有文件的 `FileObject.file_name`
  - 若目标位置已存在同名文件或文件夹，返回冲突

## 2. 递归删除非空目录

- 接口扩展：`DELETE /folders?path=...&recursive=true`
- 行为：
  - 将目标目录及其子目录统一标记为目录回收站状态
  - 将目录下所有活跃文件统一移入文件回收站
  - 若目录本身只是由文件路径派生、尚无显式 `Folder` 记录，则先补建目录元数据，再执行删除

## 3. 文件夹回收站

- 新接口：
  - `GET /recycle/folders`
  - `POST /recycle/folders/restore`
  - `DELETE /recycle/folders/purge`
- 行为：
  - `GET` 返回已删除文件夹
  - `restore` 恢复目录本身、子目录以及目录下已删除文件
  - `purge` 彻底删除目录元数据，并清理目录下已删除文件对象和元数据

## 4. 前端交互

- 文件列表页左侧文件树：
  - 新增 `重命名` 按钮
  - `删除当前文件夹` 改为支持递归删除非空目录
  - 删除确认文案明确说明“会将目录及其中内容移入回收站”

- 回收站页：
  - 新增“文件夹回收站”区域
  - 支持：
    - 恢复文件夹
    - 彻底删除文件夹

## 5. 文件树布局改造

- 目标：
  - 文件树节点文本和按钮全部横向显示，避免中文被挤成竖排
  - 同层级目录之间有清晰的虚线结构

- 方案：
  - 增大左侧树区域宽度
  - 头部操作按钮改为横向工具条，不再挤在狭窄列里
  - 表格标题与操作按钮统一 `white-space: nowrap`
  - 为树节点增加虚线连接：
    - 垂直虚线表示同一父目录下的兄弟节点
    - 水平虚线表示父子连接关系
