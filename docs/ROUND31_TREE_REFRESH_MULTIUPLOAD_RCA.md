# 第三十一轮问题根因分析

## 问题一：目录树选中项在鼠标悬停时文字看不清

- 问题现象：
  - 左侧文件树中，当前选中目录本身是深色选中态。
  - 鼠标移到选中项上时，背景会变浅，文字可读性变差。
- 可能原因：
  - 普通 hover 样式覆盖了选中态样式。
  - 选中态未定义独立的 hover 样式。
- 验证步骤：
  - 检查 `frontend/src/styles/main.css` 中目录树相关样式优先级。
- 验证结果：
  - `.folder-tree-panel .p-tree-node-content:hover` 会直接把选中项背景改成浅灰。
  - 选中态只有 `[data-p-selected=\"true\"]`，没有单独定义 `:hover`。
- 最终根因：
  - 普通 hover 规则覆盖了选中态的视觉规则。
- 修复方案：
  - 为“未选中节点 hover”和“已选中节点 hover”拆分样式。
  - 选中节点 hover 保持深色体系，并提高文字对比度。

## 问题二：删除或恢复后列表会闪一下白屏

- 问题现象：
  - 文件删除、回收站恢复/彻底删除后，列表区域会闪一下明显的白色遮罩。
- 可能原因：
  - 每次动作完成后都触发整表 `loading`。
  - PrimeVue DataTable 的默认 loading mask 是整块浅色覆盖层。
- 验证步骤：
  - 检查 `FileCenterView.vue` 与 `RecycleBinView.vue` 的刷新逻辑。
  - 检查 PrimeVue 表格是否直接绑定了 `loading`。
- 验证结果：
  - 删除/恢复/彻底删除后的刷新都会复用 `loadFiles()` / `loadRecycleFiles()`，这两个方法都会把 `loading` 设置为 `true`。
  - `DataTable` 同时绑定 `:loading=\"loading\"`，所以动作后会出现整表 loading 遮罩。
- 最终根因：
  - 行级动作后的“静默刷新”被错误地实现成了“整表 loading 刷新”。
- 修复方案：
  - 为列表加载增加 `silent` 模式。
  - 查询、翻页时仍显示整表 loading。
  - 删除、恢复、彻底删除等动作完成后的刷新改为静默刷新，只更新数据，不再盖整表白罩。

## 问题三：上传页不支持多文件选择和多文件拖拽

- 问题现象：
  - 点击选择文件时只能选 1 个文件。
  - 拖入多个文件时只上传第 1 个。
- 可能原因：
  - 上传状态只有单个 `selectedFile`。
  - 拖拽和文件输入都只读取 `files[0]`。
  - 上传流程只支持单个上传会话。
- 验证步骤：
  - 检查 `frontend/src/views/UploadView.vue` 当前状态和事件处理。
- 验证结果：
  - 组件只维护 `selectedFile`。
  - `onFileChange` 与 `onDrop` 都只取第一个文件。
  - `startUpload()` 只执行一次单文件会话。
- 最终根因：
  - 上传页使用的是单文件状态机，当前实现天然不支持批量上传。
- 修复方案：
  - 将单文件状态改为文件队列。
  - 文件输入开启 `multiple`。
  - 拖拽支持批量文件。
  - 上传按钮按队列顺序逐个上传，显示总进度和逐文件结果。
